Detailed Description

The random number engine used by all distributions in pqRand.

engine wraps the seeded PRNG in an API designed for the quantile flip-flop. This provides access to both quasuniform and superuniform samples from \( U \), a random bool that doesn't waste bits, and a simple way to apply a random sign. For the most part, these tools are used internally by prRand's distributions.

Arbitrary states of pqRand_engine can be stored using WriteState() and re-seeded using Seed_FromFile(). See uPRNG_64_Seeder for more details.

Note
Future changes to the API are not anticipated, but may still be possible

Constructor & Destructor Documentation

pqRand::engine::engine ( bool const  autoSeed = true)
inline

Construct the engine; auto-seed if requested.

To auto-seed engine, construct like:

pqRand::engine gen1();
pqRand::engine gen2(true);

To supply a seed, construct like:

pqRand::engine gen3(false); // Don't auto-seed (leaving gen3 is an undefined state).
gen3.Seed_FromFile("./archive/Jan2017/run123.seed"); // Seed from a file (gen3 is no longer undefined).

If one intends to supply a user-generated seed (as opposed to storing/reusing an auto-seed), they should use the minimal state-string format (which is described in Seed()) using a state_size of 16.

Warning
engine contains more state information than the minimal state-string. This extra information should only be output by WriteState(), and never generated by the user.
Parameters
autoSeedPerform an autoSeed (true) by calling Seed() or defer seeding till later (false).
