Detailed Description

template<class prng64_t>
class pqRand::uPRNG_64_Seeder< prng64_t >

A wrapper for a 64-bit PRNG, providing a seeding interface (used by engine).

Author
Keith Pedersen (Keith.nosp@m..Dav.nosp@m.id.Pe.nosp@m.ders.nosp@m.en@gm.nosp@m.ail..nosp@m.com)

The seeding functions fill the entire state of the PRNG (because you can't properly seed a PRNG with 128 bytes of state using a 4-byte integer). The recommended method is using Seed() to "auto-seed" the PRNG.

The state of the PRNG can be converted to an ASCII "state-string" by GetState(). This state-string can be written directly to a file by WriteState(). This permits reseeding by Seed_FromString() or Seed_FromFile() (respectively). The PRNG can also be seeded from a user-generated state-string, provided that it is in the correct format (see Seed()).

Warning
Seed_FromFile() and Seed_FromString() do not check the format of state-strings before forwarding them to prng64_t.

Seed_FromStream() and WriteState_ToStream() are simple wrappers to prng64_t 's operator<< and operator>>. They are the actual worker functions called by the public Seed/State functions, and are virtual so that a derived PRNG class may add more information to its internal state (beyond the state of the underlying PRNG) without redefining the public Seed/State functions.

Parameters
prng64_tA PRNG class supplying uint64_t. It must:
  • have a default constructor
  • possess the following static fields
    static const word_size = 64; // Number of bits per PRNG word
    static const state_size; // Number of words of seed entropy required.
  • implement the following functions to input/output the state of the generator in ASCII format
    friend operator>>(std::istream&, prng64_t&);
    friend operator<<(std::ostream&, prng64_t&);
  • support the "minimal" format used by Seed().

Note that the default constructor auto-seeds. This scheme allows the best of both worlds:

  • Automatically use high-quality seed in most cases (lest you forget).
  • Allow seeding from stored states when explicitly requested (see engine::engine for an example).

template<class prng64_t>
void pqRand::uPRNG_64_Seeder< prng64_t >::Seed ( )

Auto-seed the generator using std::random_device.

std::random_device is intended to provide access to a small amount of true randomness (or at least a non-deterministic PRNG). On many GNU/Linux systems, std::random_device uses \dev\urandom. Seed() calls std::random_device until the generator's state is filled.

Seed() constructs its state-string using the "minimal" format described in xorshift1024_star::operator<<. Thus, for prng64_t = xorshift1024_star, the state variable p is not supplied, activating the default p behavior of xorshift1024_star::operator>>.
